## 数据数组  
  
上一章展示了 Python 提供了一些非常有用的且灵活的通用数据结构。特别地，`list` 对象可以被视为一个真正的主力，具有许多方便的特性和应用领域。使用这种灵活的（可变的）数据结构是有代价的，表现为相对较高的内存占用、较慢的性能，或两者兼有。然而，科学和金融应用通常需要对特定的数据结构进行高性能操作。在这方面最重要的数据结构之一就是数组。数组通常将相同数据类型的其他（基本）对象组织成行和列。  
  
暂且假设只涉及数字，尽管这一概念也同样适用于其他类型的数据。在最简单的情况下，从数学上讲，一维数组代表一个通常为实数的向量，在内部由 `float` 对象表示。它仅由单行或单列的元素组成。在更常见的情况下，数组代表一个 $i \times j$ 元素的矩阵。这个概念可以推广到三维的 $i \times j \times k$ 元素立方体，以及形状为 $i \times j \times k \times l \times ....$ 的一般 $n$ 维数组。  
  
像线性代数和向量空间理论这样的数学分支表明，这种数学结构在许多科学学科和领域中都非常重要。因此，拥有专门设计用于方便高效处理数组的特定数据结构类被证明是卓有成效的。这就是 Python 库 NumPy 发挥作用的地方，它拥有强大的 `ndarray` 类。在下一节介绍该类之前，本节首先展示两种处理数组的替代方案。  
  
### 使用 Python 列表的数组  
  
数组可以使用上一章中介绍的内置数据结构来构建。`list` 对象特别适合完成这项任务。一个简单的 `list` 已经可以被视为一个一维数组：  
  
```python  
In [1]: v = [0.5, 0.75, 1.0, 1.5, 2.0] # ❶ 带有数字的列表对象。  
```  
  
由于 `list` 对象可以包含任意其他对象，它们也可以包含其他 `list` 对象。通过这种方式，通过嵌套 `list` 对象很容易构建二维和更高维的数组：  
  
```python  
In [2]: m = [v, v, v] # ❶ 包含列表对象的列表对象...  
        m             # ❷ ...产生一个数字矩阵。  
```  
  
```txt  
Out[2]: [[0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0]]  
```  
  
人们还可以通过简单索引轻松选择行，或通过双重索引选择单个元素（然而，整列并不那么容易选择）：  
  
```python  
In [3]: m[1]  
```  
  
```txt  
Out[3]: [0.5, 0.75, 1.0, 1.5, 2.0]  
```  
  
```python  
In [4]: m[1][0]  
```  
  
```txt  
Out[4]: 0.5  
```  
  
可以进一步嵌套以获得更通用的结构：  
  
```python  
In [5]: v1 = [0.5, 1.5]  
        v2 = [1, 2]  
        m = [v1, v2]  
        c = [m, m] # ❶ 数字立方体。  
        c  
```  
  
```txt  
Out[5]: [[[0.5, 1.5], [1, 2]], [[0.5, 1.5], [1, 2]]]  
```  
  
```python  
In [6]: c[1][1][0]  
```  
  
```txt  
Out[6]: 1  
```  
  
请注意，以刚刚介绍的方式组合对象通常通过指向原始对象的引用指针进行。这在实践中意味着什么？来看看以下操作：  
  
```python  
In [7]: v = [0.5, 0.75, 1.0, 1.5, 2.0]  
        m = [v, v, v]  
        m  
```  
  
```txt  
Out[7]: [[0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0]]  
```  
  
现在更改 `v` 对象的第一个元素的值，看看 `m` 对象会发生什么：  
  
```python  
In [8]: v[0] = 'Python'  
        m  
```  
  
```txt  
Out[8]: [['Python', 0.75, 1.0, 1.5, 2.0],  
 ['Python', 0.75, 1.0, 1.5, 2.0],  
 ['Python', 0.75, 1.0, 1.5, 2.0]]  
```  
  
可以使用 `copy` 模块的 `deepcopy()` 函数来避免这种情况：  
  
```python  
In [9]: from copy import deepcopy  
        v = [0.5, 0.75, 1.0, 1.5, 2.0]  
        m = 3 * [deepcopy(v), ] # ❶ 使用物理副本代替引用指针。  
        m  
```  
  
```txt  
Out[9]: [[0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0]]  
```  
  
```python  
In [10]: v[0] = 'Python' # ❷ 因此，原始对象的更改...  
         m               # ❸ ...不再有任何影响。  
```  
  
```txt  
Out[10]: [[0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0],  
 [0.5, 0.75, 1.0, 1.5, 2.0]]  
```  
  
### Python array 类  
  
Python 中有一个专用的 `array` 模块可用。根据文档说明：  
  
> 该模块定义了一种对象类型，它可以紧凑地表示基本值：字符、整数、浮点数的数组。数组是序列类型，其行为与列表非常相似，只是存储在其中的对象类型受到约束。该类型在创建对象时使用类型码指定，类型码是单个字符。  
  
考虑以下代码，它从 `list` 对象实例化了一个 `array` 对象：  
  
```python  
In [11]: v = [0.5, 0.75, 1.0, 1.5, 2.0]  
  
In [12]: import array  
  
In [13]: a = array.array('f', v) # ❶ 使用 float 作为类型码实例化 array 对象。  
         a  
```  
  
```txt  
Out[13]: array('f', [0.5, 0.75, 1.0, 1.5, 2.0])  
```  
  
```python  
In [14]: a.append(0.5) # ❷ 主要方法的工作方式类似于 list 对象的方法。  
         a  
```  
  
```txt  
Out[14]: array('f', [0.5, 0.75, 1.0, 1.5, 2.0, 0.5])  
```  
  
```python  
In [15]: a.extend([5.0, 6.75]) # ❷ 主要方法的工作方式类似于 list 对象的方法。  
         a  
```  
  
```txt  
Out[15]: array('f', [0.5, 0.75, 1.0, 1.5, 2.0, 0.5, 5.0, 6.75])  
```  
  
```python  
In [16]: 2 * a # ❸ 虽然“标量乘法”在原则上有效，但结果并不是数学上预期的那样；相反，元素被重复了。
```  


尝试附加与指定数据类型不同的对象会引发 `TypeError`：

## 常规 NumPy 数组  
  
使用 `list` 对象组成数组结构在某种程度上是可行的。但它并不是很方便，而且 `list` 类并非为了这个特定目标而建立。它的范围要广泛和通用得多。`array` 类稍微专门化一些，提供了一些处理数据数组的有用功能。然而，一个真正专门化的类对于处理数组类型结构来说才是非常有益的。

### 多维度  
  
向多维度的过渡是无缝的，并且迄今为止介绍的所有特性都可以继承到更一般的情况。特别是，索引系统在所有维度上保持一致：  
  
```python  
In [53]: b = np.array([a, a * 2]) # ❶ 从一维 ndarray 对象构造一个二维 ndarray 对象。  
         b  
```  
  
```txt  
Out[53]: array([[ 0.,  1.,  2.,  3.,  4.,  5.,  6.,  7.],  
       [ 0.,  2.,  4.,  6.,  8., 10., 12., 14.]])
```  

```python  
In [54]: b[0] # ❷ 选择第一行。  
```  
  
```txt  
Out[54]: array([0., 1., 2., 3., 4., 5., 6., 7.])  
```  
  
```python  
In [55]: b[0, 2] # ❸ 选择第一行中的第三个元素；索引在括号内用逗号分隔。  
```  
  
```txt  
Out[55]: 2.0  
```  
  
```python  
In [56]: b[:, 1] # ❹ 选择第二列。
```  

## 结构化 NumPy 数组  
  
`ndarray` 类的特殊性显然带来了许多有价值的好处。然而，如果专门化程度太高，对于大多数基于数组的算法和应用程序来说可能会成为过重的负担。因此，NumPy 提供了结构化的 `ndarray` 对象和记录 `recarray` 对象，允许你每列使用不同的 `dtype`。“每列”是什么意思？考虑以下对结构化 `ndarray` 对象的初始化：  
  
```python  
In [118]: dt = np.dtype([('Name', 'S10'), ('Age', 'i4'),  
                         ('Height', 'f'), ('Children/Pets', 'i4', 2)]) # ❶ 组合的复杂 dtype。  
  
In [119]: dt # ❶ 组合的复杂 dtype。  
```  
  
```txt  
Out[119]: dtype([('Name', 'S10'), ('Age', '<i4'), ('Height', '<f4'),  
       ('Children/Pets', '<i4', (2,))])  
```  
  
```python  
In [120]: dt = np.dtype({'names': ['Name', 'Age', 'Height', 'Children/Pets'],  
                         'formats':'O int float int,int'.split()}) # ❷ 达到相同结果的替代语法。  
  
In [121]: dt # ❷ 达到相同结果的替代语法。  
```  
  
```txt  
Out[121]: dtype([('Name', 'O'), ('Age', '<i8'), ('Height', '<f8'),  
       ('Children/Pets', [('f0', '<i8'), ('f1', '<i8')])])  
```

> **结构化数组**  
>  
> 除了常规数组，NumPy 还提供结构化（和记录）数组，允许描述和处理具有每个（命名）列不同数据类型的类表格数据结构。它们将类似 SQL 表的数据结构引入 Python，并具有常规 `ndarray` 对象的大部分优势（语法、方法、性能）。  
  
## 代码向量化  
  
向量化是一种获得更紧凑、执行速度可能更快代码的策略。其基本思想是“一次性”对复杂对象进行运算或应用函数，而不是循环遍历对象的各个元素。在 Python 中，像 `map()` 和 `filter()` 这样的函数式编程工具提供了一些向量化的基本手段。然而，NumPy 在其核心深处内置了向量化机制。  
  
### 基本向量化

NumPy 所做的是只需对对象逐个元素应用函数 `f`。从这种意义上讲，通过使用此类操作，人们并没有避免循环；只是避免了在 Python 层面的循环，并将循环委托给了 NumPy。在 NumPy 层面，通过优化的代码处理循环遍历 `ndarray` 对象的操作，其中大部分代码是用 C 语言编写的，因此通常比纯 Python 代码要快。这也解释了使用 NumPy 获得基于数组场景性能优势背后的“秘密”。  
  
### 内存布局  
  
当使用 `np.zeros()` 初始化 `ndarray` 对象时，正如在第 94 页的“多维度”中那样，它提供了一个用于内存布局的可选参数。这个参数大概来说是指定数组中的哪些元素在内存中相邻（连续）存储。在处理小型数组时，这对数组操作的性能几乎没有任何可衡量的影响。但是，当数组变大，并且取决于要在其上实现的（金融）算法时，情况可能就不同了。这就是内存布局发挥作用的时候（参见，例如 Eli Bendersky 的文章 “多维数组的内存布局”）。  
  
为了说明数组内存在科学和金融中布局的潜在重要性，请思考以下多维 `ndarray` 对象的构建：  
  
```python  
In [149]: x = np.random.standard_normal((1000000, 5)) # ❶ 两个维度存在很大不对称性的大型 ndarray 对象。  
  
In [150]: y = 2 * x + 3 # ❷ 对原始对象数据进行线性变换。  
  
In [151]: C = np.array((x, y), order='C') # ❸ 创建一个 C 顺序（行优先）的二维 ndarray 对象。  
  
In [152]: F = np.array((x, y), order='F') # ❹ 创建一个 F 顺序（列优先）的二维 ndarray 对象。  
  
In [153]: x = 0.0; y = 0.0 # ❺ 释放内存（取决于垃圾回收）。  
  
In [154]: C[:2].round(2) # ❻ 来自 C 对象的一些数字。  
```

* 对于 F 顺序（列优先）的 `ndarray` 对象，与按行求和相比，按列求和相对较快。  
  
## 结论  
  
NumPy 是 Python 中进行数值计算的首选包。`ndarray` 类专门设计用于方便、高效地处理（大量）数值数据。强大的方法和 NumPy 通用函数允许使用向量化代码，在很大程度上避免了 Python 层面缓慢的循环。本章介绍的许多方法同样适用于 pandas 及其 `DataFrame` 类（参见第 5 章）。
