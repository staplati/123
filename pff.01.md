## Python编程语言  
  
Python是一种高级的、多用途的编程语言，被广泛应用于各个领域和技术部门。在Python网站上你可以找到以下执行摘要：  
  
Python是一种解释型、面向对象、具有动态语义的高级编程语言。它的高级内置数据结构，结合动态类型和动态绑定，使其对于快速应用开发非常具有吸引力，同时也非常适合作为脚本语言或胶水语言来连接现有的组件。Python简单、易学的语法强调可读性，从而降低了程序维护的成本。Python支持模块和包，这鼓励了程序的模块化和代码重用。Python解释器和广泛的标准库以源码或二进制形式免费提供给所有主要平台，并且可以自由分发。  
  
这段话很好地描述了为什么Python能演变成当今主流编程语言之一。如今，无论是初学编程的人，还是技术高超的专家开发者；无论是在学校、大学、互联网公司，还是在大型企业和金融机构，抑或是任何科学领域，都在使用Python。  
  
在其他特性中，Python具有以下特点：  
  
**开源**  
Python以及绝大多数可用的支持库和工具都是开源的，并且通常带有非常灵活和开放的许可证。  
  
**解释型**  
参考的CPython实现是该语言的解释器，它在运行时将Python代码转换为可执行的字节码。  
  
**多范式**  
Python支持不同的编程和实现范式，例如面向对象编程以及命令式、函数式或过程式编程。  
  
**多用途**  
Python可用于快速的交互式代码开发以及构建大型应用程序；它既可用于底层系统操作，也可用于高级分析任务。  
  
**跨平台**  
Python可用于大多数重要的操作系统，如Windows、Linux和macOS。它被用于构建桌面以及Web应用程序，并且可以在最大的集群和最强大的服务器上使用，也可以在像Raspberry Pi这样的小型设备上使用。  
  
**动态类型**  
Python中的类型通常是在运行时推断出来的，而不是像大多数编译型语言那样静态声明的。  
  
**缩进感知**  
与大多数其他编程语言相比，Python使用缩进而不是括号、方括号或分号来标记代码块。  
  
**垃圾回收**  
Python具有自动垃圾回收功能，使程序员无需手动管理内存。  
  
当谈到Python语法和Python的本质时，Python增强提案（PEP）20——即所谓的“Python之禅”——提供了主要的指导原则。在每个交互式shell中都可以通过 `import this` 命令访问它：

import this

## Python生态系统  
  
与仅仅作为一种编程语言相比，Python作为一个生态系统的一个主要特点是拥有大量的包和工具可用。这些包和工具通常需要在需要时被导入（例如，一个绘图库），或者必须作为一个单独的系统进程启动（例如，一个Python交互式开发环境）。导入意味着使一个包对当前的命名空间和当前的Python解释器进程可用。  
  
Python本身已经附带了大量的包和模块，这些包和模块在不同方向上增强了基础解释器的功能，被称为Python标准库。例如，基本的数学计算可以在不导入任何内容的情况下完成，而更专业的数学函数则需要通过math模块导入：  
  
```python  
100 * 2.5 + 50  
```  
  
```txt  
300.0  
```  
  
```python  
# ❶ 如果没有进一步导入，会引发错误。  
log(1)   
```  
  
```txt  
-----------------------------------------------------------------  
NameError Traceback (most recent call last)  
<ipython-input-3-74f22a2fd43b> in <module>  
----> 1 log(1)   
NameError: name 'log' is not defined  
```  
  
```python  
import math   
# ❷ 导入math模块后，可以执行该计算。  
math.log(1)   
```  
  
```txt  
0.0  
```  
  
虽然math是一个包含在任何Python安装中的标准Python模块，但还有许多包可以选装，并且可以像使用标准模块一样使用。此类包可从不同的（网络）来源获得。不过，通常建议使用Python包管理器，以确保所有库之间相互兼容（有关此主题的更多信息，请参见第2章）。  
  
到目前为止呈现的代码示例分别使用了交互式Python环境：IPython和Jupyter。在撰写本文时，它们可能是使用最广泛的交互式Python环境。虽然IPython最初只是一个增强的交互式Python shell，但如今它拥有许多通常在集成开发环境（IDE）中才能找到的特性，例如支持性能分析和调试。IPython中缺失的那些特性通常由高级的文本/代码编辑器提供，比如Vim，它也可以与IPython集成。因此，将IPython与自己选择的文本/代码编辑器结合起来，形成Python开发过程的基础工具链，并不罕见。  
  
IPython在许多方面增强了标准的交互式shell。除其他外，它提供了改进的命令行历史功能，并允许轻松进行对象检查。例如，只需在函数名前后加上一个 `?` 即可打印出该函数的帮助文本（docstring）（添加 `??` 将提供更多信息）。  
  
IPython最初有两个流行的版本：shell版本和基于浏览器的版本（Notebook）。Notebook变体被证明非常有用且受欢迎，以至于它演变成了一个独立的、与语言无关的项目，现在称为Jupyter。在此背景下，Jupyter Notebook继承了IPython的大多数有益特性并不令人意外——并且它提供了更多功能，例如在数据可视化方面。  
  
有关使用IPython的更多详细信息，请参阅VanderPlas（2016，第1章）。  
  
## Python用户群体  
  
Python不仅吸引了专业的软件开发者；它对临时开发者以及领域专家和科学开发者也很有用。  
  
**专业软件开发者** 会在Python中找到高效构建大型应用程序所需的一切。它支持几乎所有的编程范式；有强大的开发工具可用；并且原则上任何任务都可以用Python来解决。这类用户通常会构建自己的框架和类，也会在基础Python和科学计算栈上进行开发，并努力充分利用生态系统。  
  
**科学开发者** 或 **领域专家** 通常是某些包和框架的重度用户，他们构建了自己的应用程序，并随着时间的推移对其进行增强和优化，以及根据自身特定需求定制生态系统。这些用户群体通常也会参与较长时间的交互式会话，快速进行新代码的原型设计，以及探索和可视化他们的研究和/或领域数据集。  
  
**临时程序员** 通常喜欢将Python用于他们知道Python擅长处理的特定问题。例如，访问matplotlib的图库页面，复制那里提供的某段可视化代码，并根据他们的特定需求调整代码，对于这个群体的成员来说可能就是一个有益的使用场景。  
  
还有另一个重要的Python用户群体：**初学者程序员**，即那些刚刚开始编程的人。如今，Python已经成为大学、学院甚至学校中非常流行的用来向学生介绍编程的语言。原因之一是其基本语法易于学习和理解，即使对于非开发者也是如此。此外，Python支持几乎所有的编程风格，这也是有帮助的。

## 科学计算栈  
  
有一组特定的包被统称为科学计算栈。该包栈主要包括以下几个包：  
  
**NumPy**  
NumPy提供了一个多维数组对象，用于存储同质或异质数据；它还提供了优化的函数/方法来对该数组对象进行操作。  
  
**SciPy**  
SciPy是实现了科学或金融领域经常需要的重要标准功能的子包和函数的集合；例如，可以找到用于三次样条插值以及数值积分的函数。  
  
**matplotlib**  
这是Python最受欢迎的绘图和可视化包，提供2D和3D的可视化能力。  
  
**pandas**  
pandas建立在NumPy之上，提供了更丰富的类来管理和分析时间序列和表格数据；它与matplotlib和PyTables紧密集成，用于绘图以及数据的存储和检索。  
  
**scikit-learn**  
scikit-learn是一个流行的机器学习（ML）包，为许多不同的ML算法（例如估计、分类或聚类）提供了一个统一的应用程序编程接口（API）。  
  
**PyTables**  
PyTables是HDF5数据存储包的一个流行包装器；它是一个基于分层数据库/文件格式实现优化的、基于磁盘的I/O操作的包。  
  
根据特定领域或问题的不同，这个包栈会被额外的包所扩充，这些包通常建立在这些基础包中的一个或多个之上。然而，总体的最小公分母或基本构建块是NumPy的ndarray类（见第4章）和pandas的DataFrame类（见第5章）。  
  
仅将Python视为一种编程语言，其实也有其他几种可用语言在语法和优雅性上能够与之媲美。例如，Ruby就是一种经常被拿来与Python比较的流行语言。该语言的网站将Ruby描述为：  
  
> 一种专注于简单性和生产力的动态、开源编程语言。它拥有优雅的语法，阅读自然且编写容易。  
  
大多数使用Python的人可能也会认同将完全相同的评价用于Python本身。然而，对于许多用户来说，Python区别于Ruby等同样吸引人的语言的地方在于科学计算栈的可用性。这使得Python不仅是一种好用且优雅的语言，而且是一种能够取代特定领域语言和工具集（如Matlab或R）的语言。它还默认提供了你所期望的一切，比如作为一个经验丰富的Web开发者或系统管理员所需要的工具。此外，Python擅长与特定领域语言（如R）进行交互，因此通常的决定不是在Python或其他某语言之间二选一——而是在于哪种语言应该作为主要语言。  
  
## 金融科技  
  
在对Python的本质有了这些“粗略概念”之后，我们有必要稍微退一步，简要思考一下科技在金融领域的角色。这将使我们能够更好地判断Python已经发挥的作用，更重要的是，它将在未来金融行业中可能扮演的角色。  
  
在某种意义上，科技本身对金融机构（比如与生物技术公司相比）或金融功能（比如与物流等其他企业功能相比）来说并没有什么特别之处。然而，近年来，在创新和监管的刺激下，银行和对冲基金等其他金融机构越来越向科技公司演变，而不再仅仅是金融中介。科技已经成为全球几乎所有金融机构的主要资产，有可能带来竞争优势，也可能带来竞争劣势。一些背景信息可以阐明这种发展背后的原因。  
  
### 科技支出  
  
银行和金融机构合计构成了每年在科技上支出最多的行业。因此，以下声明不仅表明科技对金融行业很重要，而且金融行业对科技部门也同样非常重要：  
  
> 马萨诸塞州弗雷明汉，2018年6月14日 – 根据国际数据公司（IDC）一系列金融服务IT支出指南的新数据，到2021年，全球金融服务公司的信息技术（IT）支出将达到近5000亿美元，较2018年的4400亿美元有所增长。  
> ——IDC  
  
特别是，银行和其他金融机构正竞相使其业务和运营模式数字化：  
  
> 2017年，北美地区银行在新技术上的支出预计将达到199亿美元。  
> 银行开发现有系统并致力于新的技术解决方案，以提高其在全球市场的竞争力，并吸引对新在线和移动技术感兴趣的客户。这对于为银行业提供新思路和软件解决方案的全球金融科技公司来说是一个巨大的机遇。  
> ——Statista  
  
如今，大型跨国银行通常雇佣数千名开发人员来维护现有系统并构建新系统。对技术要求很高的大型投资银行每年的技术预算通常高达数十亿美元。  
  
### 科技作为赋能者  
  
技术发展也促进了金融领域的创新和效率提升。通常，该领域的项目都在数字化的保护伞下进行。  
  
> 过去几年，金融服务行业经历了由技术主导的急剧变化。许多高管指望他们的IT部门来提高效率并促进改变游戏规则的创新——同时还能以某种方式降低成本并继续支持原有系统。与此同时，金融科技初创企业正在侵蚀既定市场，凭借从头开始开发的、不受原有系统羁绊的客户友好型解决方案占据领先地位。  
> ——普华永道第19届全球CEO年度调查 2016  
  
随着效率提高所带来的副作用，竞争优势往往必须在日益复杂的产品或交易中寻找。这反过来必然增加风险，使风险管理以及监督和监管变得越来越困难。2007年和2008年的金融危机讲述了此类发展可能带来的潜在危险。同样，“算法和计算机失控”代表了金融市场的潜在风险；这在2010年5月所谓的闪电崩盘中戏剧性地体现出来，当时自动化抛售导致某些股票和股票指数在盘中大幅下跌。第四部分涵盖了与金融工具算法交易相关的主题。  
  
### 科技与人才是进入壁垒  
  
一方面，在其他条件不变的情况下，技术进步会随着时间的推移降低成本。另一方面，金融机构继续在技术上大量投资，以赢得市场份额并捍卫现有地位。如今，要在金融的某些领域保持活跃，往往需要同时在技术和熟练员工方面进行大规模投资。以衍生品分析领域为例：  
  
> 汇总整个软件生命周期来看，采用内部策略进行OTC（场外衍生品）定价的公司，仅构建、维护和增强一个完整的衍生品库就需要2500万美元至3600万美元之间的投资。  
> ——Ding (2010)  
  
构建一个成熟的衍生品分析库不仅成本高昂且耗时，而且你还需要有足够的专家来完成这项工作。而这些专家必须拥有合适的工具和技术来完成他们的任务。随着Python生态系统的发展，现在这种努力变得更加高效，与大约10年前相比，这方面的预算可以大幅削减。第五部分涵盖了衍生品分析，并仅利用Python和标准的Python包构建了一个小巧但强大且灵活的衍生品定价库。  
  
另一段关于长期资本管理公司（LTCM）早期情况的引用——它曾经是最受尊敬的量化对冲基金之一，但在20世纪90年代末破产了——进一步证实了这种关于技术和人才的见解：  
  
> Meriwether斥资2000万美元购买了一套最先进的计算机系统，并聘请了一支由金融工程师组成的精英团队来运营在康涅狄格州格林威治设立的LTCM。这是一种工业级别的风险管理。  
> ——Patterson (2010)  
  
Meriwether过去需要花数百万美元购买的计算能力，今天也许几千美元就可以买到，或者可以基于灵活的付费计划从云提供商那里租用。第2章将展示如何在云中搭建基础设施，以便使用Python进行交互式金融分析、应用程序开发和部署。这样一套专业的基础设施预算每月仅从几美元起。另一方面，对于大型金融机构而言，如今的交易、定价和风险管理已经变得如此复杂，以至于它们需要部署包含数以万计计算核心的IT基础设施。  
  
### 不断增长的速度、频率和数据量  
  
金融行业受技术进步影响最大的一个维度是决定和执行金融交易的速度和频率。Lewis (2014) 生动地描述了所谓的高频交易（flash trading）——即以尽可能高的速度进行交易。  
  
一方面，在越来越小的时间尺度上数据可用性的增加，使得我们需要实时做出反应。另一方面，交易速度和频率的增加又进一步推动了数据量的增加。这就导致了相互强化的过程，并系统性地拉低了金融交易的平均时间尺度。这个趋势在十年前就已经开始了：  
  
> 文艺复兴科技公司（Renaissance）的大奖章基金（Medallion fund）在2008年获得了惊人的80%的收益，利用其快如闪电的计算机从市场的极端波动中获利。吉姆·西蒙斯（Jim Simons）成为当年对冲基金界的最高收入者，把诱人的25亿美元收入囊中。  
> ——Patterson (2010)  
  
一支单只股票30年的每日股票价格数据大约代表了7500个收盘报价。今天的大多数金融理论正是基于这类数据建立的。例如，现代（均值-方差）投资组合理论（MPT）、资本资产定价模型（CAPM）和风险价值（VaR），都奠基在每日股票价格数据之上。  
  
相比之下，在一个典型的交易日的单个交易小时内，苹果公司（AAPL）的股票价格可能会被报价约15,000次——这大约是30年来可用的日末收盘报价数量的两倍（见第24页“数据驱动与AI优先的金融”中的例子）。这带来了一系列挑战：  
  
**数据处理**  
仅仅考虑和处理股票或其他金融工具的日末报价是不够的；因为在一天中发生的事情“太多了”，对某些工具来说，甚至是一周7天、每天24小时都在发生变化。  
  
**分析速度**  
决策通常必须在几毫秒甚至更短的时间内做出，这使得建立相应的分析能力并实时分析大量数据成为必要。  
  
**理论基础**  
尽管传统的金融理论和概念远非完美，但它们随着时间的推移已经得到了很好的检验（有时也被很好地驳斥了）；而对于今天至关重要的毫秒和微秒尺度而言，在传统意义上随时间证明有一定鲁棒性的一致金融概念和理论，仍然缺失。  
  
所有这些挑战通常只能通过现代技术来解决。可能令人有些惊讶的是，一致性理论缺乏的问题往往通过技术手段来解决，因为高速算法利用的是市场微观结构元素（例如订单流、买卖价差），而不是依赖于某种金融逻辑推理。  
  
## 实时分析的兴起  
  
在金融行业中，有一门学科的重要性显著上升：金融与数据分析。这一现象与行业内速度、频率和数据量急剧增加的洞察有着密切关系。事实上，实时分析可以被视为该行业对这一趋势的应对之策。  
  
粗略地说，“金融与数据分析”指的是结合（可能很高级的）算法和方法，应用软件和技术来收集、处理和分析数据，以获取洞察、做出决策或满足监管要求等目的的学科。例子可能包括估计银行零售分行某项金融产品定价结构变化带来的销售影响，或者为一家投资银行复杂衍生品交易投资组合大规模隔夜计算信用估值调整（CVA）。  
  
金融机构在此背景下面临两个主要挑战：  
  
**大数据**  
甚至在“大数据”这个词被创造出来之前，银行和其他金融机构就必须处理海量数据；然而，在单次分析任务中需要处理的数据量随着时间的推移急剧增加，这要求提高计算能力，以及不断扩大的内存和存储容量。  
  
**实时经济**  
在过去，决策者可以依赖结构化的、定期的规划以及决策和（风险）管理流程，而今天他们面临着需要实时处理这些功能的局面；过去在后台通过隔夜批处理运行的几项任务，现在已经被转移到前台并实时执行。  
  
同样，我们可以观察到技术进步与金融/业务实践之间的相互作用。一方面，存在着通过应用现代技术不断提高分析方法在速度和能力方面水平的需求。另一方面，技术方面的进步允许采用新的分析方法，而这些方法在几年前甚至几个月前被认为是不可能的（或由于预算限制而不可行）。  
  
在分析领域的一个主要趋势是：在中央处理器（CPU）方面利用并行架构，以及在通用图形处理器（GPGPU）方面利用大规模并行架构。目前的GPGPU拥有数以千计的计算核心，有时需要从根本上重新思考对不同算法而言并行的含义是什么。在这方面，仍然存在的障碍是，用户通常必须学习新的编程范式和技术来驾驭这种硬件的威力。

### 金融与Python语法  
  
在金融背景下迈出使用Python第一步的大多数人，可能都会着手解决一个算法问题。这类似于一个科学家，比方说，想要解一个微分方程、计算一个积分，或者只是简单地将一些数据可视化。一般来说，在这个阶段，很少有人会去考虑像正式开发流程、测试、文档或部署等话题。然而，这恰恰是人们爱上Python的阶段。这背后的一个主要原因可能是Python语法通常非常接近用于描述科学问题或金融算法的数学语法。  
  
这可以通过一个金融算法来说明，即通过蒙特卡洛模拟对欧洲看涨期权进行估值。该例子考虑的是Black-Scholes-Merton（BSM）模型设定，在该设定下，期权的标的风险因子遵循几何布朗运动。  
  
假设以下用于估值的数值参数值：  
  
* 初始股票指数水平 $S_0 = 100$  
* 欧洲看涨期权执行价格 $K = 105$  
* 到期时间 $T = 1$ 年  
* 恒定无风险短期利率 $r = 0.05$  
* 恒定波动率 $\sigma = 0.2$  
  
在BSM模型中，到期时的指数水平是一个随机变量，由等式1-1给出，其中 $z$ 是一个标准正态分布的随机变量。  
  
等式 1-1. Black-Scholes-Merton (1973) 到期指数水平  
  
$$S_T = S_0 \exp\left(\left(r - \frac{1}{2}\sigma^2\right)T + \sigma \sqrt{T} z\right)$$  
  
以下是蒙特卡洛估值程序的算法描述：  
  
1. 从标准正态分布中提取 $I$ 个伪随机数 $z(i)$ ， $i \in \{1, 2, ..., I\}$ 。  
2. 给定 $z(i)$ 和等式1-1，计算所有由此产生的到期指数水平 $S_T(i)$ 。  
3. 计算期权到期时的所有内在价值为 $h_T(i) = \max(S_T(i) - K, 0)$ 。  
4. 通过等式1-2给出的蒙特卡洛估计量来估计期权现值。  
  
等式 1-2. 欧洲期权的蒙特卡洛估计量  
  
$$C_0 \approx e^{-rT} \frac{1}{I} \sum_{i} h_T(i)$$  
  
这个数学问题和算法现在必须转换为Python。以下代码实现了所需的步骤：  
  
```python  
import math  
# ❶ NumPy在这里被用作主要的包。  
import numpy as np   
  
# ❷ 定义模型和模拟参数值。  
S0 = 100.   
K = 105.   
T = 1.0   
r = 0.05   
sigma = 0.2   
  
I = 100000   
  
# ❸ 固定随机数生成器的种子值。  
np.random.seed(1000)   
  
# ❹ 提取标准正态分布的随机数。  
z = np.random.standard_normal(I)   
  
# ❺ 模拟期末数值。  
ST = S0 * np.exp((r - sigma ** 2 / 2) * T + sigma * math.sqrt(T) * z)   
  
# ❻ 计算到期时的期权收益。  
hT = np.maximum(ST - K, 0)   
  
# ❼ 评估蒙特卡洛估计量。  
C0 = math.exp(-r * T) * np.mean(hT)   
  
# ❽ 打印最终的价值估计结果。  
print('Value of the European call option: {:5.3f}.'.format(C0))   
```  
  
```txt  
Value of the European call option: 8.019.  
```  
  
有三个方面值得强调：  
  
**语法**  
Python语法的确非常接近数学语法，例如在分配参数值时。  
  
**翻译**  
每个数学和/或算法语句通常都可以被翻译成单行Python代码。  
  
**向量化**  
NumPy的优势之一是紧凑的向量化语法，例如，允许在单行代码内完成100,000次计算。  
  
这段代码可以在像IPython或Jupyter Notebook这样的交互式环境中使用。然而，需要定期重用的代码通常会被组织成所谓的模块（或脚本），也就是后缀为 .py 的单一Python文件（技术上说是文本文件）。在这种情况下，这样一个模块看起来就像例子1-1，并且可以被保存为名为 bsm_mcs_euro.py 的文件。  
  
例子 1-1. 欧洲看涨期权的蒙特卡洛估值  
  
```python  
#  
# 欧洲看涨期权的蒙特卡洛估值  
# 在 Black-Scholes-Merton 模型中  
# bsm_mcs_euro.py  
#  
# 金融Python，第2版  
# (c) Dr. Yves J. Hilpisch  
#  
import math  
import numpy as np  
  
# 参数值  
S0 = 100. # 初始指数水平  
K = 105. # 执行价格  
T = 1.0 # 到期时间  
r = 0.05 # 无风险短期利率  
sigma = 0.2 # 波动率  
  
I = 100000 # 模拟次数  
  
# 估值算法  
z = np.random.standard_normal(I) # 伪随机数  
# 到期指数水平  
ST = S0 * np.exp((r - 0.5 * sigma ** 2) * T + sigma * math.sqrt(T) * z)  
hT = np.maximum(ST - K, 0) # 到期收益  
C0 = math.exp(-r * T) * np.mean(hT) # 蒙特卡洛估计量  
  
# 结果输出  
print('Value of the European call option %5.3f.' % C0)  
```  
  
本小节中的算法例子说明，Python凭借其语法，非常适合作为科学语言经典组合（英语和数学）的补充。似乎将Python加入科学语言体系，使其变得更加全面。因此我们有：  
  
* **英语** 用于书写和讨论科学和金融问题等。  
* **数学** 用于简明扼要、准确地描述和建模抽象概念、算法、复杂数量等。  
* **Python** 用于在技术层面建模并实现抽象概念、算法、复杂数量等。  
  
> 数学与Python语法  
> 几乎没有任何一种编程语言能像Python这样接近数学语法。因此，通常将数值算法从数学表示形式直接翻译为Python实现是非常简单的。这使得在金融领域使用Python进行原型设计、开发和代码维护变得非常高效。  
  
在某些领域，使用伪代码（pseudo-code）是很常见的做法，以此引入第四位语言家族成员。伪代码的作用是以更技术化的方式来表示金融算法，这种方式既接近数学表示形式，又很接近技术实现。除了算法本身，伪代码还会考虑到计算机工作原理的基本原则。  
  
这种做法通常是因为在大多数（编译型）编程语言中，技术实现与其正式的数学表示往往相差甚远。大部分编程语言要求包含太多仅从技术角度需要的元素，使得很难看出数学公式与代码之间的等价性。  
  
如今，Python经常被当做伪代码来使用，因为它的语法几乎与数学等价，而且其技术上的“开销”被降至最低。这是通过该语言中包含的许多高级概念来实现的，这些概念不仅有其优势，通常也伴随着风险和/或其他成本。然而，可以稳妥地说，当你需要时，完全可以使用Python遵循其他语言可能从一开始就要求的同等严格的实现和编码规范。从这个意义上讲，Python可以提供两全其美的结果：高度的抽象与严格的实现。

### 确保高性能  
  
一般来说，人们公认Python拥有相当简洁的语法，用它编写代码相对高效。然而，正是由于Python作为解释型语言的性质，很多人一直存在这样一种偏见：认为Python在金融计算密集型任务中往往太慢。的确，根据特定的实现方法，Python有时确实非常慢。但它并非注定如此——它在几乎任何应用领域都可以表现出很高的性能。原则上，为了获得更好的性能，至少可以区分出三种不同的策略：  
  
**惯用法和范式**  
一般而言，在Python中许多不同的方法都可以得到相同的结果，但有时它们的性能特征却大相径庭；“仅仅”选择正确的方法（例如，特定的实现方式，如明智地使用数据结构，通过向量化避免循环，或使用像pandas这样的特定包）可以显著改善结果。  
  
**编译**  
如今，有几种性能包可用，它们提供了重要函数的编译版本，或者可以静态或动态（在运行时或调用时）将Python代码编译为机器码，这可以使这些函数比纯Python代码快几个数量级；其中比较流行的是Cython和Numba。  
  
**并行化**  
许多计算任务，特别是在金融领域，可以从并行执行中显著获益；这对于Python来说并不是什么新鲜事，而是可以很容易利用它来实现的。  
  
> 使用Python进行高性能计算  
> Python本身并不是一项高性能计算技术。然而，Python已经发展成为一个可以访问当前高性能计算技术的理想平台。从这个意义上说，Python已经成为类似高性能计算技术胶水语言的角色。  
  
本小节采用一个简单但依旧贴合实际的例子，该例子涉及了所有三种策略（后面的章节将详细说明这些策略）。金融分析中一个相当常见的任务是在大型数字数组上评估复杂的数学表达式。为此，Python本身就提供了所需的一切：  
  
```python  
import math  
loops = 2500000  
a = range(1, loops)  
def f(x):  
    return 3 * math.log(x) + math.cos(x) ** 2  
%timeit r = [f(x) for x in a]  
```  
  
```txt  
1.59 s ± 41.2 ms per loop (mean ± std. dev. of 7 runs, 1 loop each)  
```  
  
在这种情况下，Python解释器大约需要1.6秒来对 `f()` 函数求值2,500,000次。相同的任务可以使用NumPy来实现，它提供了优化过（即预编译）的函数来处理这类基于数组的操作：  
  
```python  
import numpy as np  
a = np.arange(1, loops)  
%timeit r = 3 * np.log(a) + np.cos(a) ** 2  
```  
  
```txt  
87.9 ms ± 1.73 ms per loop (mean ± std. dev. of 7 runs, 10 loops each)  
```  
  
使用NumPy将执行时间大幅减少到了大约88毫秒。不过，还有一个专门针对此类任务的包。它叫做numexpr，代表“数值表达式（numerical expressions）”。它会编译表达式以提升常规NumPy功能的性能，比如避免在此过程中出现ndarray对象的内存内拷贝：  
  
```python  
import numexpr as ne  
ne.set_num_threads(1)  
f = '3 * log(a) + cos(a) ** 2'  
%timeit r = ne.evaluate(f)  
```  
  
```txt  
50.6 ms ± 4.2 ms per loop (mean ± std. dev. of 7 runs, 10 loops each)  
```  
  
使用这种更专业的方法进一步将执行时间减少到了大约50毫秒。然而，numexpr还具备内置的并行执行相应操作的能力。这使得我们可以利用CPU的多个线程：  
  
```python  
ne.set_num_threads(4)  
%timeit r = ne.evaluate(f)  
```  
  
```txt  
22.8 ms ± 1.76 ms per loop (mean ± std. dev. of 7 runs, 10 loops each)  
```  
  
并行化使执行时间在这个例子中（利用了四个线程）进一步降至23毫秒以下。总体而言，这代表着性能提升了90倍以上。特别要注意的是，这种改进是在不改变基本问题/算法、也不需要掌握编译或并行化方法的任何细节的情况下实现的。即使是非专家，也能在高级层面上使用这些功能。当然，前提是你需要了解有哪些功能和选项可用。  
  
这个例子表明，Python提供了多种选项来更充分地利用现有资源——也就是说，提高生产力。使用并行方法，相比于顺序执行，可以在相同的时间内完成三倍的计算量——在这个例子中，仅仅是告诉Python使用多个可用的CPU线程而不是仅仅使用一个。  
  
## 从原型设计到生产环境  
  
交互式分析中的效率和执行速度方面的性能显然是考虑Python的两个好处。然而，将Python用于金融领域的另一个主要好处乍看之下似乎不那么明显；但仔细一想，它可能会展现为金融机构的一个重要战略因素。那就是端到端地使用Python的可能性——从原型设计一直到生产环境。  
  
如今，纵观全球金融机构，在金融开发流程方面的实践仍然常常被描述为一个分离的两步流程。一方面，有负责模型开发和技术原型设计的量化分析师（"quants"）。他们喜欢使用Matlab和R这类允许快速、交互式应用开发的工具和环境。在开发的这个阶段，诸如性能、稳定性、部署、访问管理和版本控制等问题并不那么重要。人们主要寻找的是能展示算法或整个应用程序主要所需特征的概念验证和/或原型。  
  
一旦原型完成，IT部门的开发者接手并负责将现有的原型代码转化为可靠、可维护且高性能的生产代码。通常，在这个阶段会发生范式转换，即使用编译型语言（如C++或Java）来满足部署和生产环境的要求。同时，也会普遍应用带有专业工具、版本控制等的正式开发流程。  
  
这种两步法通常会产生许多意想不到的后果：  
  
**效率低下**  
原型代码不可重用；算法必须实现两次；冗余的工作消耗时间和资源；翻译转换过程中会产生风险。  
  
**技能集不同**  
不同部门展现出不同的技能集，并使用不同的语言来实现“同样的事情”；大家不仅在编程上有差异，而且使用的是不同的“语言”进行交流。  
  
**遗留代码**  
遗留的代码不仅依然存在，还必须在不同语言中维护，通常采用了不同的实现风格。  
  
另一方面，使用Python能够实现一个简化的端到端流程：从最初的交互式原型设计直到高度可靠且易于维护的生产代码。不同部门之间的沟通变得更加容易。员工培训也变得更加精简，因为只有一种涵盖金融应用程序构建各个领域的主要语言。这还避免了在开发流程的不同阶段使用不同技术所带来的内在低效和冗余。总而言之，Python可以为金融分析、金融应用开发和算法实现中几乎所有的任务提供一致的技术框架。  
  
## 数据驱动与AI优先的金融  
  
本书第一版在2014年首次阐述了有关技术与金融行业关系的几乎所有观察，在2018年8月为本书第二版更新本章时，这些观点似乎仍然相当时新和重要。然而，本节将评论正在根本性地重塑金融行业的两大趋势。这两大趋势主要在过去几年中凸显出来。  
  
### 数据驱动金融  
  
一些最重要的金融理论，如MPT和CAPM，可以追溯到20世纪50年代和60年代。然而，它们仍然代表了经济学、金融学、金融工程和工商管理等领域学生教育的基石。这可能令人感到意外，因为对大多数这些理论的经验支持充其量也就非常有限，而且证据往往与理论的暗示和含义完全相反。另一方面，它们的流行也是可以理解的，因为它们接近人们对金融市场如何运作的预期，并且它们是建立在一些令人吸引——即便通常过于简单——的假设之上的优雅数学理论。  
  
科学方法（比如说在物理学中）是从数据开始的，例如从实验或观察中获取数据，进而提出假设和理论，然后根据数据对它们进行测试。如果测试结果是积极的，假设和理论可能会被完善并正式写下来，例如，以供发表的学术论文形式。如果测试结果是消极的，那么假设和理论会被拒绝，研究人员将重新开始寻找符合数据的假设和理论。既然物理定律随时间稳定，一旦发现这种定律并经过良好验证，最好的情况下它将永远存在。  
  
大型机构的（量化）金融历史在很大程度上与科学方法相矛盾。在很多情况下，理论和模型是“从零开始”建立在简化数学假设之上的，目的是为金融核心问题寻找优雅的答案。比如，在金融中一个常见的假设是金融工具收益服从正态分布，以及目标数量之间存在线性关系。由于这些现象在金融市场中几乎从未出现过，因此对于这些优雅理论往往缺乏实证证据也就不足为奇了。许多金融理论和模型首先被提出、证明并发表，后来才受到实证检验。在一定程度上，这当然是因为在1950年代到1970年代，乃至更晚的时期，连如今金融系本科生所能轻易获取的金融数据，在当时都是无法获得的。  
  
自20世纪90年代初中期以来，金融机构对这类数据的获取能力急剧增加，如今甚至从事金融研究或参与算法交易的个人也能通过流媒体服务访问到巨量的历史刻度级（tick level）数据以及实时刻度数据。这使得我们可以回归科学方法：通常先有数据，然后才设计想法、假设、模型和策略。

在Python中最广泛使用的机器学习包之一是 `scikit-learn`。接下来的代码展示了如何（以一种极度简化的方式）利用ML中的分类算法来预测未来市场价格变动方向，并基于此预测建立一种算法交易策略。所有的细节将在第15章进行解释，因此这里的例子非常简练。首先，数据的导入与特征数据的准备（带有方向的滞后对数收益率数据）：  
  
```python  
import numpy as np  
import pandas as pd  
  
# ❶ 选取苹果公司股票(AAPL.O)的历史日末数据。  
data = pd.read_csv('../../source/tr_eikon_eod_data.csv',  
                   index_col=0, parse_dates=True)  
data = pd.DataFrame(data['AAPL.O'])   
# ❷ 计算全历史时间段的对数收益率。  
data['Returns'] = np.log(data / data.shift())   
data.dropna(inplace=True)  
  
lags = 6  
cols = []  
for lag in range(1, lags + 1):  
    col = 'lag_{}'.format(lag)  
    # ❸ 生成包含定向滞后对数收益数据的DataFrame列（+1或-1）。  
    data[col] = np.sign(data['Returns'].shift(lag))   
    cols.append(col)  
data.dropna(inplace=True)  
```  
  
接下来是为支持向量机（SVM）算法实例化一个模型对象、拟合模型以及执行预测步骤。图1-2显示：这种基于预测的交易策略（根据预测做多或做空苹果公司的股票），跑赢了针对该股票自身的被动基准投资。  
  
```python  
from sklearn.svm import SVC  
  
# ❶ 实例化模型对象。  
model = SVC(gamma='auto')   
  
# ❷ 给定特征和标签数据（全部为方向性的），拟合模型。  
model.fit(data[cols], np.sign(data['Returns']))   
```

  decision_function_shape='ovr', degree=3, gamma='auto', kernel='rbf',

  tol=0.001, verbose=False)

```python  
# ❸ 使用拟合好的模型生成预测结果（样本内），同时这也是交易策略的仓位（做多或做空）。  
data['Prediction'] = model.predict(data[cols])   
  
# ❹ 给定预测值和基准对数收益率，计算交易策略的对数收益率。  
data['Strategy'] = data['Prediction'] * data['Returns']   
  
# ❺ 绘制基于ML的交易策略与被动基准投资的性能对比图。  
data[['Returns', 'Strategy']].cumsum().apply(np.exp).plot(  
                                            figsize=(10, 6));   
```
